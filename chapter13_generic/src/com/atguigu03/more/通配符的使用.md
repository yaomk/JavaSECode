## 通配符 `<?>` 的使用

| 形式            | 含义                    | 适用场景                 |
|:--------------|:----------------------|:---------------------|
| `?`           | 无界通配符，表示任意类型          | 不需要知道具体类型，只做与类型无关的操作 |
| `? extends T` | 上界通配符，表示类型是 T 或 T 的子类 | 从泛型对象中获取数据（生产者）      |
| `? super T`   | 下界通配符，表示类型是 T 或 T 的父类 | 向泛型对象中存入数据（消费者）      |

### 使用说明

举例：`ArrayList<?>`

`ClassA<?>` 可以看作 `ClassA<T>` 类型的父类，即可以将 `ClassA<T>` 的对象赋值给 `ClassA<?>` 类型的引用（或变量）。

### 读写数据的特点（以集合 `ArrayList<?>` 为例）

* 读取数据：允许读取数据，读取的值的类型为 `Object` 类型。
* 写入数据：不允许写入数据。特例：可以写入 `null` 值。

### 有限制条件的通配符

可以将 java 中的类简单理解为数学中从左到右的坐标轴，坐标的最左侧为 `null`，最右侧为 `Object`。<br />
`? extends T` 可以简单理解为 `<=` 。<br />
`? super T` 可以简单理解为 `>=` 。

* 上界通配符 `? extends T`
    * 表示类型是 `T` 或 `T` 的子类，适合生产者场景：从中读取数据。

```java
public double sum(List<? extends Number> numbers) {
    double sum = 0.0;
    for (Number n : numbers) {   // 可以安全地读取为 Number
        sum += n.doubleValue();
    }
    // numbers.add(123);    // 编译错误！不能添加，因为具体子类型未知
    return sum;
}

public static void test() {
    // 调用
    List<Integer> ints = Arrays.asList(1, 2, 3);
    List<Double> doubles = Arrays.asList(1.1, 2.2);
    System.out.println(sum(ints));   // 6.0
    System.out.println(sum(doubles)); // 3.3
}
```

* 下界通配符 `? super T`
    * 表示类型是 T 或 T 的父类，适合消费者场景：向其中存入数据。

```java
public void addNumbers(List<? super Integer> list) {
    list.add(1);
    list.add(2);
    // Integer i = list.get(0); // 编译错误！读取只能得到 Object
    Object obj = list.get(0);    // 可以读取为 Object
}

public static void test() {
    // 调用
    List<Number> numbers = new ArrayList<>();
    List<Object> objects = new ArrayList<>();
    addNumbers(numbers);   // 合法，Number 是 Integer 的父类
    addNumbers(objects);   // 合法
    // List<Integer> ints = ...; addNumbers(ints); // 错误，Integer 不是 Integer 的父类
}
```
