## 泛型在继承上的体现

### `类 SuperA` 是 `类 A` 的父类，则 `G<SuperA>` 与 `G<A>` 的关系：

* `G<SuperA>` 和 `G<A>` 是并列的两个类，没有任何子父类的关系。
* 泛型之间没有继承关系，不能使用 `is a` 关系。

举例：

```text
ArrayList<Object> list1 = null;
ArrayList<String> list2 = null;

// list1 = list2; // 编译报错！
```

* 如果允许这样赋值会发生什么？ <br/>
  假设 Java 允许 `ArrayList<Object> list = new ArrayList<String>();`，那么就会出现类型安全问题：

```text
list = list1;          // 假设合法
list.add(123);         // 添加一个 Integer 对象到 ArrayList<Object> 引用中
String s = list1.get(0); // 期望 String，实际得到 Integer → ClassCastException
```

### `类 SuperA` 是 `类 A` 的父类或接口，则 `SuperA<T>` 与 `A<T>` 的关系：

* `SuperA<T>` 与 `A<T>` 有继承或实现的关系。
* 即 `A<T>` 的实例可以赋值给 `SuperA<T>` 类型的引用（或变量）。

举例：

```text
List<String> list1 = null;
ArrayList<String> list2 = new ArrayList<>();

list1 = list2; // 编译通过
```
