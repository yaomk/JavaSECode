## 自定义泛型类、接口

### 格式

```java
class A<T> {

}

interface B<T1, T2> {

}
```

### 使用说明

1. 我们在声明完自定义泛型类后，可以在类的内部（比如：属性、方法、构造器中）使用类的泛型。
2. 我们在创建自定义泛型类的对象时，可以指明泛型参数类型。一旦声明，内部凡是使用类的泛型参数的位置，都具体化为指定的类的泛型类型。
3. 如果在创建自定义泛型类的对象时，没有指明泛型参数类型，那么泛型将被擦除，泛型对应的类型均按照 `Object` 处理，但不等价于
   `Object`。
    * 经验：泛型要使用一路都用。要不用，一路都不用。
    * 原始类型与 `Class<Object>` 在编译器的类型检查、赋值兼容性、方法解析等方面有本质区别。原始类型彻底抛弃了泛型信息，是“退化的”泛型，主要用于兼容旧代码；而
      `Class<Object>` 仍然是完整的参数化类型，保留泛型规则。
4. 泛型的指定中必须使用引用数据类型。不能使用基本数据类型，此时只能使用包装类替换。
5. 除创建泛型类对象外，子类继承泛型类时、实现泛型类接口时，也可以确定泛型结构中的泛型参数。如果我们在给泛型类提供子类时，子类也不确定泛型的类型，则可以继续使用泛型参数。还可以在现有的父类的泛型参数的基础上，新增泛型参数。

### 注意

1. 泛型类可能有多个参数，此时应将多个参数一起放在尖括号内。比如：`<T1, T2, T3>`
2. JDK7.0 开始，泛型的简化操作：`ArrayList<String> list = new ArrayList<>();`
3. 如果泛型结构是一个接口或抽象类，则不可创建泛型类的对象。
4. 不能使用 `new E[]`。但是可以使用类型强转：`E[] elements = (E[]) new Object[capacity];`
    * 参考：`ArrayList` 源码中声明：`Object[] elementData`，而非泛型参数类型数组。
5. 在类、接口上声明的泛型，在本类或本接口中即代表类实例、子类或实现类的某种类型，不可以在静态方法中使用类的泛型。
    * 类的泛型参数在创建实例时才确定，静态方法在类加载时存在，无法获知具体类型。
6. 异常类不能是带泛型的。

## 自定义泛型方法

* 问题：在泛型类的方法中，使用了类的泛型参数。那么此方式是泛型方法吗？不是！

```java
public class GenericClass<T> {
    public T test(T t) {
    } // 不是泛型方法

    public <E> E GenericMethod(E e) {
    } // 是泛型方法
}
```

### 格式

```text
权限修饰符 <泛型参数列表> 返回值类型 方法名(参数列表) {

}
// 通常在形参列表或返回值类型的位置会出现泛型类型
public <E> E GenericMethod(E e) {}
```

### 使用说明

1. 在声明泛型方法时，一定要添加泛型参数 `<T>`。
2. 泛型参数在方法调用时，指明其具体类型。
3. 泛型方法可以根据需要声明为静态（static）的。
4. 泛型方法跟所在的类是否是一个泛型类没有关系。
